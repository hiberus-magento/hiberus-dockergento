### Data-Driven Image Configuration (#298)

Moved `nginx`, `mailhog`, `rabbitmq`, and `hitch` image references from hardcoded lines in `docker-compose.template.yml` into `data/requirements.json` placeholders, consistent with all other services. Image resolution now uses a `:` heuristic тАФ values containing `:` are used verbatim (e.g. `mariadb:11.4`, `rabbitmq:4.1-management`); plain tags are prefixed with `hiberusmagento/<service>:`.

### Dynamic Compatibility Table (#298)

`hm compatibility` now generates the version table dynamically from `data/requirements.json` and `data/equivalent_versions.json`. No hardcoded version strings remain in the script тАФ new versions appear automatically as they are added to the data files.

## ЁЯРЫ Bug Fixes

### `hm create-project` Ghost Directory on macOS (#298)

Fixed a macOS Docker Desktop bug where missing bind-mount source paths (`composer.json`, `composer.lock`) were created as empty directories inside the container instead of files. This caused `cp: cannot overwrite directory with non-directory` errors when Composer tried to write its output.

**Fix**: Pre-create `composer.json` and `composer.lock` on the host with `touch` before `docker compose up`. Docker Desktop then mounts them as files, not directories.

## ЁЯФз Improvements

### Speckit Integration

Added Speckit slash commands and extensions for all supported AI platforms (Claude Code, OpenCode, GitHub Copilot/Agents, Cursor, Gemini). Enables AI-assisted specification, planning, implementation, and iteration workflows directly from the editor.

### `ai-registration.json` as Team-Shared State (#298)

`config/docker/ai-registration.json` is now tracked in version control, enabling a team-wide AI tools state:

- **Architect**: runs `hm ai-init` + `hm ai-pull` тЖТ commits `ai-properties.json` + `ai-registration.json`
- **Team members**: run `hm ai-pull` after `git pull` to sync AI tools to the registered versions

Paths stored in the registration file are relative, making it portable across machines.

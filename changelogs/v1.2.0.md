# Changelog v1.2.0 (2026-04-01)

## 🚀 New Features

### AI Tools Management System (#XXX)

Complete implementation of AI tools management system for Hiberus Dockergento CLI, enabling automated download and management of AI coding assistant skills and agents.

#### New Commands

**`hm ai-init`** - Initialize AI Tools Configuration
- Interactive wizard for configuring AI platforms, skill types, and resources
- Multi-select menus for platforms: claude, cursor, codex, copilot, gemini, opencode
- Multi-select for skill types: hyva, acs, magento, php
- Resource selection: skills, agents, or both
- Non-interactive mode with command-line flags (`--platforms`, `--types`, `--resources`)
- Custom repository support (`--repository`, `--branch`)
- Pre-fill support for reconfiguration (reads existing configuration)
- Auto-triggers download after configuration
- Creates `config/docker/ai-properties.json` (team-shared, committable)

**`hm ai-pull`** - Update AI Tools
- Downloads/updates skills and agents from configured repositories
- Reads configuration from `config/docker/ai-properties.json`
- Preserves custom skills/agents using whitelist approach
- `--force` flag to override custom file protection
- Continues on repository failures (resilient to unavailable repos)
- Updates `config/docker/ai-registration.json` with tracked files
- Supports both default and custom repositories

**`hm ai-reset`** - Remove Downloaded Tools
- Removes auto-downloaded skills/agents while preserving custom ones
- Reads `config/docker/ai-registration.json` for tracked files
- Interactive confirmation prompt (or `--confirm` flag for automation)
- Whitelist-based deletion (only removes tracked files)
- Shows summary of preserved custom items
- Clears registration but preserves configuration
- Refuses to operate if registration file is corrupted

#### Key Features

**Extensible Platform System**
- JSON-based platform definitions in `data/ai-platforms.json`
- Easy addition of new AI platforms without code changes
- Supports: Claude Code, Cursor, GitHub Copilot, Codex, Gemini, OpenCode
- Platform-specific directory conventions (`.claude/skills`, `.cursor/agents`, etc.)

**Extensible Skill Type System**
- JSON-based skill type definitions in `data/ai-skill-types.json`
- Easy categorization: hyva, acs (Adobe Commerce Storefront), magento, php
- Filterable downloads based on project needs

**Repository Management**
- Default repositories defined in `data/ai-repositories.json`
- Custom repository support per project
- GitHub Archive API for tarball downloads (4-10x faster than git clone)
- Automatic fallback to git clone if tarball fails
- Retry logic with 3 attempts and exponential backoff
- HTTPS-only validation for security

**Configuration System**
- Two-file approach:
  - `ai-properties.json`: Team-shared configuration (committed to VCS)
  - `ai-registration.json`: Local state tracking (gitignored)
- Configuration hierarchy: runtime args → project config → tool defaults
- Atomic write operations with validation
- SHA256 checksums for integrity tracking

**Custom Skills/Agents Protection**
- Whitelist-based approach: only tracked files can be deleted
- Custom skills/agents coexist with downloaded ones
- Preserved during updates and resets
- `--force` flag available when override needed

- Team adoption flow: ~14 seconds (target: <60s) ✓

#### Use Cases

**Initial Setup**
```bash
# Interactive wizard
hm ai-init

# Non-interactive
hm ai-init --platforms=claude,cursor --types=hyva,magento --resources=skills,agents
```

**Team Adoption**
```bash
# New team member with existing config
hm ai-pull

# Updates pulled from configured repositories automatically
```

**Update Tools**
```bash
# Update when repositories have new versions
hm ai-pull

# Force overwrite (including custom)
hm ai-pull --force
```

**Reset/Clean**
```bash
# Remove downloaded tools, keep custom
hm ai-reset

# Skip confirmation
hm ai-reset --confirm
```

**Custom Repositories**
```bash
# Add company-internal repositories
hm ai-init --repository=https://github.com/company/ai-tools --branch=main
```

**Reconfiguration**
```bash
# Run ai-init again to modify configuration
# Wizard pre-fills existing selections
hm ai-init
```

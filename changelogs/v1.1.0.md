# Changelog v1.1.0 (2026-03-31)

## 🚀 New Features

### Magento 2.4.7 Support (#280)
- Added support for Magento 2.4.7
- PHP 8.3 support
- OpenSearch 2.12 support  
- Redis 7.2 support
- Updated requirements matrix for version compatibility

### Docker Compose v2 Support (#296)
- Automatic detection of Docker Compose v1/v2
- Full backward compatibility with `docker-compose` (v1)
- Centralized command execution via `$DOCKER_COMPOSE` variable
- Priority: `docker compose` (v2) → `docker-compose` (v1) fallback
- Improved error handling and container validation

### Architecture Documentation (#292)
- Added comprehensive architecture documentation
- Segmented documentation structure (ARCHITECTURE.md)
- CLI architecture, Docker orchestration, and command translation docs
- State management and configuration hierarchy documentation

## 🐛 Bug Fixes

### Security Version Support (#294)
- Fixed support for Magento versions with security patches (e.g., 2.4.6-p3)
- Separate `MAGENTO_VERSION` (full) and `EQUIVALENT_VERSION` (base) variables
- Proper version handling in `create-project` command
- Resolved issue #281

### Command Execution Improvements (#293)
- Improved argument handling in mysql commands
- Enhanced transfer-db and transfer-media operations
- Better error handling and validation

## 🔧 Improvements

### Docker Compose YAML Optimization (#295)
- Conditional `version` field in docker-compose files
- Removes obsolete `version` attribute for Docker Compose >= 2.25
- Eliminates warning messages in modern Docker Compose versions
- Shared `version_gte()` helper for semantic version comparison
- Resolves issue #283

## 📚 Technical Improvements

- Extracted `version_gte()` to shared helper (`console/helpers/version.sh`)
- Refactored version comparison logic (DRY principle)
- Enhanced security with container validation in SSL operations
- Fixed parameter expansion vulnerabilities in exec.sh
- Improved POSIX compliance (newlines at EOF)

## 🔄 Backward Compatibility

All changes maintain full backward compatibility:
- Docker Compose v1 continues working seamlessly
- Existing Magento versions (2.3.x - 2.4.6) unaffected
- No breaking changes in CLI commands or configuration

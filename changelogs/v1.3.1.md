## 📝 Technical Notes

### Bash Parameter Default with Braces
When using parameter defaults like `${var:-default}`, if `var` contains braces (e.g., `{}`), bash concatenates causing `${1:-{}}` to produce `{}}`  when `$1="{}"`. Use explicit checks instead:
```bash
local config="$1"
if [[ -z "${config}" ]]; then
    config="{}"
fi
```

### Stdout vs Stderr Best Practices
Functions that return values via command substitution must write all UI output to stderr:
```bash
print_message() {
    echo "User message" >&2  # To stderr
    echo "$return_value"    # To stdout (captured)
}
```

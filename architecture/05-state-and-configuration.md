## 5. State and Configuration Management

### 5.1 Configuration Hierarchy

The tool uses a **multi-layered** configuration system that allows per-project customization:

```
1. Global Configuration (~/hm/data/)
   ↓ Overridden by...
2. Project Configuration (config/docker/)
   ↓ Overridden by...
3. Runtime Environment Variables
```

### 5.2 Configuration Files

#### 5.2.1 Global Configuration (`~/hm/data/`)

**`data/properties.json`** - System default properties:
```json
{
    "MAGENTO_DIR": ".",
    "BIN_DIR": "./vendor/bin",
    "HOST_DIR": ".",
    "DOCKER_COMPOSE_FILE": "docker-compose.yml",
    "DOCKER_COMPOSE_FILE_LINUX": "docker-compose.dev.linux.yml",
    "DOCKER_COMPOSE_FILE_MAC": "docker-compose.dev.mac.yml",
    "USER_PHP": "app",
    "GROUP_PHP": "app",
    "WORKDIR_PHP": "/var/www/html",
    "COMPOSE_PROJECT_NAME": ""
}
```

**`data/requirements.json`** - Magento → Services compatibility matrix:
```json
{
    "2.4.7": {
        "php": "8.3-bookworm",
        "mariadb": "10.6",
        "search": "2.12-opensearch",
        "redis": "7.2",
        "varnish": "7.1",
        "composer": "2.7"
    }
}
```

**`data/config.json`** - Magento installation configuration:
```json
{
    "language": "es_ES",
    "currency": "EUR",
    "timezone": "Europe/Madrid",
    "admin-firstname": "hiberus",
    "admin-user": "hiberus",
    "admin-password": "Hiberus123"
}
```

**`data/local_settings.json`** - Magento configurations for development:
```json
{
    "config_set": [
        {
            "path": "web/unsecure/base_url",
            "value": "URL"  // Will be replaced by https://$DOMAIN/
        },
        {
            "path": "catalog/search/elasticsearch7_server_hostname",
            "value": "search"  // Docker service name
        },
        {
            "path": "admin/security/session_lifetime",
            "value": "31536000"  // 1 year for development
        }
    ]
}
```

**`data/command_descriptions.json`** - Command metadata (used by help system)

**`data/equivalent_versions.json`** - Mapping of equivalent versions for compatibility

**`data/default_files_magento.json`** - List of files that must be copied between host and container

#### 5.2.2 Project Configuration (`<project>/config/docker/`)

This configuration is **project-specific** and is created during `hm setup`:

**`config/docker/properties.json`** - Overrides global properties:
```json
{
    "MAGENTO_DIR": "./src",
    "COMPOSE_PROJECT_NAME": "myproject",
    "DOMAIN": "myproject.local"
}
```

**`config/docker/local_settings.json`** - Editable copy of Magento configurations
- Created by copying from `data/local_settings.json`
- Can be modified by developers for project-specific configurations

**Typical structure**:
```
project/
├── config/
│   └── docker/
│       ├── properties.json          # Project-specific properties
│       ├── local_settings.json      # Project Magento configurations
│       └── masquerade/              # Anonymization configuration
│           └── magento2/
│               └── custom.yaml
├── docker-compose.yml               # Generated by the tool
├── docker-compose.dev.mac.yml       # macOS overlay
├── docker-compose.dev.linux.yml     # Linux overlay
└── src/                             # Magento code (if MAGENTO_DIR="./src")
```

### 5.3 Configuration Loading Process

#### 5.3.1 Initialization Flow

```bash
# bin/run → init_dirs() → import_dependencies()
1. source "$TASKS_DIR"/load_properties.sh
     ↓
2. refactor_old_version()  # Converts old format if it exists
     ↓
3. load_colors()  # Defines color variables for UI
     ↓
4. load_properties()
     ↓
     a. Read data/properties.json
     b. If config/docker/properties.json exists, read and merge
     c. Convert JSON → Environment variables with jq
     d. set -a to auto-export
```

**Merge implementation** (`console/tasks/load_properties.sh`):

```bash
if [[ -f "$CUSTOM_PROPERTIES_DIR"/properties.json ]]; then
    # Merge both JSON files
    jq -r 'to_entries[] | .key + "=\"" + .value + "\""' \
        "$DATA_DIR"/properties.json \
        "$CUSTOM_PROPERTIES_DIR"/properties.json
else
    # Only global properties
    jq -r 'to_entries[] | .key + "=\"" + .value + "\""' \
        "$DATA_DIR"/properties.json
fi | eval  # Execute to set variables
```

**Result**: Globally available variables:
```bash
export MAGENTO_DIR="./src"
export COMPOSE_PROJECT_NAME="myproject"
export DOMAIN="myproject.local"
export WORKDIR_PHP="/var/www/html"
export BIN_DIR="./vendor/bin"
# etc.
```

#### 5.3.2 Operating System Detection

```bash
# console/tasks/set_machine_specific_properties.sh
uname -s
  ↓
"Darwin" → export MACHINE="mac"
           export DOCKER_COMPOSE_FILE_MACHINE="docker-compose.dev.mac.yml"
  ↓
"Linux"  → export MACHINE="linux"
           export DOCKER_COMPOSE_FILE_MACHINE="docker-compose.dev.linux.yml"
```

**Building `$DOCKER_COMPOSE`**:
```bash
# In bin/run
DOCKER_COMPOSE="docker-compose -f $DOCKER_COMPOSE_FILE -f $DOCKER_COMPOSE_FILE_MACHINE"
# Result on Mac:
# docker-compose -f docker-compose.yml -f docker-compose.dev.mac.yml
```

### 5.4 System State Variables

#### 5.4.1 Key Environment Variables

| Variable | Origin | Purpose | Example |
|----------|--------|---------|---------|
| `COMMAND_BIN_NAME` | bin/run | CLI command name | "hm" |
| `COMMAND_TOOLNAME` | bin/run | Full tool name | "Hiberus Dockergento" |
| `COMMAND_BIN_DIR` | bin/run | Tool root directory | "~/hm" |
| `COMMANDS_DIR` | bin/run | Commands directory | "~/hm/console/commands" |
| `DATA_DIR` | bin/run | Data directory | "~/hm/data" |
| `MAGENTO_DIR` | properties.json | Path to Magento code | "." or "./src" |
| `DOMAIN` | properties.json | Local domain | "myproject.local" |
| `COMPOSE_PROJECT_NAME` | properties.json | Docker project name | "myproject" |
| `MACHINE` | set_machine_specific_properties.sh | Operating system | "mac" or "linux" |
| `DOCKER_COMPOSE` | bin/run | Full docker-compose command | "docker-compose -f docker-compose.yml -f docker-compose.dev.mac.yml" |
| `WORKDIR_PHP` | properties.json | Working directory in container | "/var/www/html" |
| `USER_PHP` | properties.json | User in PHP container | "app" |
| `REQUIREMENTS` | version_manager.sh | JSON with service versions | '{"php":"8.2-buster",...}' |
| `MAGENTO_VERSION` | input_info.sh | Selected Magento version | "2.4.6" |
| `MAGENTO_EDITION` | input_info.sh | Magento edition | "community" or "enterprise" |
| `XDEBUG_HOST` | set_xdebug_host_property.sh | Host IP for xDebug | "host.docker.internal" |
| `USE_DEFAULT_SETTINGS` | Command arguments | Use default settings | true/false |

### 5.5 Configuration Persistence

#### 5.5.1 Saving Project Properties

**Function**: `save_properties()` in `console/helpers/properties.sh`

```bash
# During hm setup
save_properties
  ↓
1. Create config/docker/ if it doesn't exist
2. Create config/docker/properties.json if it doesn't exist
3. Save key variables:
   jq -n \
     --arg MAGENTO_DIR "$MAGENTO_DIR" \
     --arg COMPOSE_PROJECT_NAME "$COMPOSE_PROJECT_NAME" \
     --arg DOMAIN "$DOMAIN" \
   '$ARGS.named' > config/docker/properties.json
```

**Persisted variables**:
- `MAGENTO_DIR`: Path to Magento directory
- `COMPOSE_PROJECT_NAME`: Project name (used to name containers)
- `DOMAIN`: Project's local domain

#### 5.5.2 Domain Detection from Database

**Function**: `set_current_domain()` in `console/tasks/set_magento_configs.sh`

```bash
# If DOMAIN is not defined, try to extract from database
exist_magento_db=$(query "SHOW TABLES LIKE 'core_config_data';")

if [[ -n $exist_magento_db && -z ${DOMAIN+x} ]]; then
    url=$(query "SELECT value FROM core_config_data WHERE path='web/secure/base_url';")
    DOMAIN=$(awk -F/ '{print $3}' <<< $url)  # Extract domain from URL
fi
```

**Usage**: When importing a database, the tool can automatically detect the configured domain.

#### 5.5.3 Docker Compose File Generation

```bash
# console/tasks/write_from_docker-compose_templates.sh
1. Read $REQUIREMENTS (JSON with versions)
2. Build substitution regex for each service
3. Apply sed over template:
   sed 's/<php_version>/8.2-buster/g; s/<mariadb_version>/10.6/g; ...' \
       docker-compose/docker-compose.template.yml > docker-compose.yml
4. Copy OS-specific overlays
```

**Persisted state**: docker-compose.yml files in the project

### 5.6 Dynamic Magento Configuration

#### 5.6.1 Applying Configurations

**Function**: `set_settings_for_develop()` in `console/tasks/set_magento_configs.sh`

```bash
# Executed after DB import or installation
1. Read config/docker/local_settings.json
2. Replace placeholders:
   - "URL" → "https://$DOMAIN/"
   - "DOMAIN" → "$DOMAIN"
3. Execute hm magento config:set for each entry
```

**Transformation example**:
```json
// local_settings.json
{"path": "web/unsecure/base_url", "value": "URL"}
  ↓
// Executed command
hm magento config:set web/unsecure/base_url https://myproject.local/
```

**Typical configurations applied**:
- Base URLs (http/https)
- Search hostname (Elasticsearch/OpenSearch)
- Admin security configurations (lifetime, password requirements)
- Cookie domain

#### 5.6.2 Extracting Domains from Database

**Function**: `get_domains.sh`

```bash
# Read all domains configured in Magento
for DOMAIN in `hm mysql -q "SELECT DISTINCT value FROM core_config_data WHERE path like 'web/%/base_url'"`
do
  if [[ "$DOMAIN" == *"://"* ]]; then
    DOMAINS+=($(echo "$DOMAIN" | sed -e 's|^[^/]*//||' -e 's|/.*$||'))
  fi
done
```

**Usage**: Commands like `hm ssl` and `hm set-host` can operate on all configured domains.

### 5.7 Docker Services State

#### 5.7.1 State Validation

The tool **does not maintain its own state of containers**, but queries Docker directly:

```bash
# Check if service is running
is_run_service "phpfpm"
  ↓
container_id=$(docker ps -qf name="$COMPOSE_PROJECT_NAME"-"phpfpm")
  ↓
if [ -z "$container_id" ]; then
    ERROR: "phpfpm service is not running!"
fi
```

**Advantage**: No desynchronization between real state and assumed state.

#### 5.7.2 Automatic Service Startup

```bash
# start_service_if_not_running.sh
$DOCKER_COMPOSE exec -T "$service" sh -c "echo 'check'" &>/dev/null
  ↓
exit code 1 → hm start
exit code 0 → Continue
```

**Use**: Commands that assume running services (e.g., `mysql`, `composer`) automatically start them if they're not active.

### 5.8 Tool-Specific Configuration

#### 5.8.1 xDebug

**Configuration**: `console/tasks/set_xdebug_host_property.sh`

```bash
# macOS: Docker Desktop provides host.docker.internal
if [[ "$MACHINE" == "mac" ]]; then
    XDEBUG_HOST="host.docker.internal"
# Linux: Get Docker gateway IP
else
    XDEBUG_HOST=$(/sbin/ip route | awk '/default/ { print $3 }')
fi

export XDEBUG_HOST="$XDEBUG_HOST"
```

**Activation/Deactivation**:
```bash
# hm debug-on
$DOCKER_COMPOSE exec phpfpm bash -c "rm /usr/local/etc/php/conf.d/docker-php-ext-xdebug.ini.disabled"
hm restart phpfpm

# hm debug-off
$DOCKER_COMPOSE exec phpfpm bash -c "mv /usr/local/etc/php/conf.d/docker-php-ext-xdebug.ini /usr/local/etc/php/conf.d/docker-php-ext-xdebug.ini.disabled"
hm restart phpfpm
```

#### 5.8.2 Masquerade (Anonymization)

**Configuration**: `config/docker/masquerade/magento2/*.yaml`

- Priority: Project custom configuration
- Fallback: Default configuration in Docker images

**Execution**:
```bash
# hm masquerade
$DOCKER_COMPOSE run --rm masquerade --config=/config magento2
```

#### 5.8.3 SSL Certificates

**Management**: `hm ssl <domain>`

```bash
# Generate certificate with mkcert on host
mkcert -key-file config/docker/ssl/$domain-key.pem \
       -cert-file config/docker/ssl/$domain.pem \
       $domain

# Copy to hitch container
docker-compose exec -T -u root hitch bash -c "cat > /etc/hitch/testcert.pem" < merged.pem
```

**Storage**: `config/docker/ssl/`

### 5.9 Migration and Compatibility

#### 5.9.1 Old Format Refactoring

**Function**: `refactor_old_version()` in `load_properties.sh`

Converts old format (key=value) to JSON:

```bash
# Old format: config/docker/properties
MAGENTO_DIR=./src
DOMAIN=myproject.local

  ↓ Conversion with jq ↓

# New format: config/docker/properties.json
{
    "MAGENTO_DIR": "./src",
    "DOMAIN": "myproject.local"
}
```

**Automatic**: Runs on every `hm` invocation if it detects the old file.
